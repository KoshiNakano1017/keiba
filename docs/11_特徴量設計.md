## 1. 特徴量とは何か

### 一言で言うと

> **AIが「答えを予測するために参照する情報のこと」**

---

### 具体例で理解する

競馬で「この馬が1着になる確率」を予測したいとき、AIは何を参考にするか？

```
❌ AIはレース映像を「見て」判断しているわけではない

✅ AIは数字に変換された情報（特徴量）を参照して判断している
```

| 生の情報 | → | 特徴量（数値化） |
|---------|---|--------------|
| 「前走は3着だった」 | → | `前走着順 = 3` |
| 「距離は1600m、芝」 | → | `距離 = 1600` `トラック種別 = 1（芝）` |
| 「1週間前の調教タイムが速かった」 | → | `坂路_直近1_4F合計 = 52.3` |
| 「単勝オッズは3.5倍」 | → | `単勝オッズ = 3.5` |
| 「前走でコーナーで詰まった（距離ロス）」 | → | `前走距離ロス = 2.5（メートル）` |

つまり特徴量とは、**「馬の強さ・レース条件・当日の状態」を数字で表したもの** です。

### なぜ特徴量が重要か

AIは数字の「パターン」から学習します。

```
よい特徴量 → AIが正確なパターンを学習できる → 高い予測精度
悪い特徴量 → ノイズになり予測精度が下がる
```

特徴量の選択と設計（特徴量エンジニアリング）は、モデルの精度を左右する **最重要工程** です。

## 2. 本件での特徴量の役割

本プロジェクトでは、1頭の馬について以下を予測します。

| 予測ターゲット | 意味 |
|-------------|------|
| 予想勝率（%） | この馬が1着になる確率 |
| 予想連対率（%） | この馬が2着以内に入る確率 |
| 予想複勝率（%） | この馬が3着以内に入る確率 |
| 期待値（単勝・複勝） | 予想確率 × 対応オッズ |

AIは「同じレースに出走する全馬の特徴量」を一緒に見て、各馬の相対的な強さを評価します。

```
レース単位の入力イメージ:
  馬A: [近走成績, 調教タイム, 騎手勝率, オッズ, 距離ロス補正, ...]
  馬B: [近走成績, 調教タイム, 騎手勝率, オッズ, 距離ロス補正, ...]
  馬C: [近走成績, 調教タイム, 騎手勝率, オッズ, 距離ロス補正, ...]
  ...
  ↓
AI が各馬のスコアを出力 → 正規化して確率（勝率・連対率・複勝率）に変換
```

## 3. CrossFactorの重要な強み

DB定義書（`08_CrossFactor_DB定義書.md`）を確認した結果、**CrossFactorはすでに多くの派生指標を計算済みで格納している** ことが判明しました。

これは通常の競馬データベースと比べて圧倒的な強みです。一般的な競馬AIは、生データから自分でこれらを計算する必要があります。

### CrossFactorが既に計算済みの指標（JV馬毎レース情報EX より）

| 指標名 | 内容 | AIでの活用 |
|-------|------|----------|
| `位置取り指数` | 過去5走のコーナー通過順位の加重平均 ÷ 出走頭数 × 100 | 脚質・位置取り傾向の把握 |
| `位置取り指数順位` | 同レース内での位置取り指数の順位 | 相対的な先行力 |
| `予想脚質` | 逃げ/先行/差し/追込の4分類 | 展開予測 |
| `4角5番手内率` | 過去レースで4コーナー5番手以内に入った割合 | 先行力の安定性 |
| `上がり3位内率` | 上がり（最後3F）が3位以内だった割合 | 末脚の安定性 |
| `PCI（ペースチェンジ指数）` | レース全体のペース評価指標 | 展開適性 |
| `休養週数` | 前走からの間隔（週数） | 状態評価 |
| `休養後何走目` | 長期休養後の出走回数 | 復帰戦の成績傾向 |
| `着度数P_馬場適性` | 芝/ダートの適性スコア | 馬場条件適性 |
| `着度数P_回り適性` | 左回り/右回りの適性スコア | コース適性 |
| `着度数P_競馬場` | 競馬場別の成績スコア | 競馬場適性 |
| `着度数P_同距離` | 同距離での成績スコア | 距離適性 |
| `競走馬単勝回収率` | 過去レースでの単勝回収率 | 期待値の安定性 |
| `競走馬複勝回収率` | 過去レースでの複勝回収率 | 期待値の安定性 |
| `平均着順_前3走` | 直近3走の平均着順 | 近走安定性 |
| `平均着順_前5走` | 直近5走の平均着順 | 中期安定性 |
| `平均人気_前3走` | 直近3走の平均人気順 | 近走の市場評価 |
| `平均人気_前5走` | 直近5走の平均人気順 | 中期の市場評価 |
| `斤量補正値` | 斤量の影響を補正した値 | 公平な能力比較 |
| `斤量補正値_上がり` | 斤量補正後の上がりタイム補正値 | 末脚評価 |
| `距離増減` | 前走からの距離変化（m） | 距離変更の影響評価 |
| `坂路_直近1〜3_各タイム` | 直近3回の坂路調教ラップタイム | 調教状態（仕上がり度） |
| `_直近1〜3_各タイム` | 直近3回のウッドチップ調教ラップ | 調教状態（距離適性） |
| `USER_指数1〜3` | ユーザー（CrossFactor）が任意に定義できるカスタム指数 | 独自評価指標 |

> **重要:** これらはすでに `JV馬毎レース情報EX` に格納されており、SQLiteから直接取得できます。  
> 自分で計算する必要がないため、**Phase 1の開発コストが大幅に削減できます。**
>
> **注:** `USER_指数1〜3` は固定の意味を持つ共通指標ではなく、各利用ユーザーが任意に定義するカスタム指数です。  
> どの `USER_指数` を採用するか、またその意味づけをどう扱うかは、実際に利用するユーザーの運用方針に合わせて決定します。

## 4. 特徴量カテゴリ一覧（MECE整理）

本件の特徴量は以下の7カテゴリに分類されます。

```
特徴量
├── A. レース条件系         ← 「このレースはどんな条件か」
├── B. 馬の能力系           ← 「この馬はどれくらい強いか」
├── C. CrossFactor独自系    ← 「競合には真似できない差別化データ」
├── D. 騎手・調教師系       ← 「人間側の強さ」
├── E. 調教系               ← 「レース直前の仕上がり状態」
├── F. オッズ・市場系       ← 「市場はどう評価しているか」
└── G. 血統系               ← 「遺伝的な適性」
```

## 6. Phase 2以降の特徴量

### Phase 2（MVP完成後）

| カテゴリ | 特徴量 | 理由 |
|---------|-------|------|
| 血統 | 父・母父の血統ID / 主要血統グループ | 距離・馬場・コース適性に遺伝的傾向がある |
| 能力の相対化 | 各特徴量の「同レース内での順位・偏差値」| 絶対値より相対順位の方が予測精度が高いケースが多い |
| 騎手詳細 | 当該馬×騎手の過去相性（合成特徴量） | 馬と騎手の相性は着順に影響する |
| ペース予測 | 推定ペース（逃げ馬の頭数・先行馬の密集度） | 展開の予測は単純な脚質以上の情報を持つ |
| 有利不利補正 | 当日のトラックバイアス（内外・前後） | CrossFactorの映像解析データが使える可能性 |

### Phase 3（深化フェーズ）

| カテゴリ | 特徴量 | 備考 |
|---------|-------|------|
| 前走の不利 | 距離ロス・進路妨害・コーナー不利 | CrossFactor映像解析データの活用。データ形式確認後に設計 |
| 走行位置詳細 | コーナーごとの位置取り変化 | 映像解析データ必要 |
| 馬場バイアス補正タイム | バイアスを除いた「純粋な走力タイム」| 独自計算が必要 |
| シーケンシャル特徴量 | 直近N走を時系列として扱う（LSTM等） | Neural Network化の際に有効 |

## 7. 特徴量エンジニアリング方針

### 7.1 欠損値の処理

| 状況 | 対処方針 |
|------|---------|
| 調教データなし（跳躍・調教なし） | `None` / `-999` で埋め、「調教なし」フラグを別途追加 |
| デビュー戦（過去実績なし） | 平均値または `初出走フラグ = 1` で対応 |
| 休養明け（長期間データなし） | `休養週数` で補完、古いデータは割引 |

### 7.2 カテゴリ変数のエンコーディング

| 変数 | エンコード方法 |
|------|-------------|
| 競馬場コード（10場） | Label Encoding（LightGBMは直接処理可能） |
| 馬場状態（良/稍重/重/不良） | 数値順序エンコード（良=0, 稍重=1, 重=2, 不良=3） |
| 予想脚質（逃/先/差/追） | Label Encoding |
| 芝/ダート | バイナリ（芝=0, ダート=1） |

### 7.3 正規化・スケーリング

LightGBMは正規化不要（決定木ベースのため）。  
ただし Neural Network（Phase 3）に移行する際は標準化が必要。

### 7.4 特徴量重要度の検証（SHAP）

```python
import shap

explainer = shap.TreeExplainer(model)
shap_values = explainer.shap_values(X_val)
shap.summary_plot(shap_values, X_val)
```

- 重要度が低い特徴量は定期的に削除してモデルをシンプルに保つ
- 「なぜその馬が高評価か」をクライアントに説明できる状態を常に維持する

## 9. データリーク防止ルール

データリークとは「未来の情報が学習データに混入してしまうこと」です。  
競馬AIでは特に注意が必要です。

### ❌ やってはいけないこと（リーク例）

| NG操作 | 理由 |
|-------|------|
| レース後のオッズを特徴量に使う | 結果が分かった後のオッズは未来情報 |
| 確定払戻金を特徴量に使う | 結果確定後の情報 |
| ランダム分割でデータを学習/検証に分ける | 未来のレースで学習して過去を予測することになる |
| 「当日の馬体重」を特日のレースに使う | 当日のレース後でないと全頭分揃わないケースがある |

### ✅ 正しい時系列分割

```
全データ
├── 学習データ    : 〜2022年12月（過去）
├── バリデーション : 2023年1月〜12月（直近1年）
└── テストデータ  : 2024年1月〜（最新・一切学習に使わない）
```

### ✅ オッズの使い方

```
使って良い : レース締め切り直前（出走確定後）の単勝オッズ
使ってはいけない : 確定着順が出た後に更新されたオッズ
```

## 参考：特徴量と予測精度の関係（先行研究より）

競馬AIの研究事例では、以下の特徴量が特に重要度が高い傾向にあります。

| 重要度ランク | 特徴量 | 理由 |
|------------|-------|------|
| 1位前後 | 単勝オッズ（人気） | 市場の集合知を反映した最強特徴量 |
| 2〜4位 | 近走成績・平均着順 | 競走馬の基礎能力 |
| 5〜8位 | 調教タイム（直近） | レース当日の仕上がり状態 |
| 9〜12位 | 騎手勝率・コース別成績 | 人的要因 |
| 13位以降 | 血統・馬場適性スコア | 条件特化型の影響 |

> ただし、CrossFactorの独自データ（位置取り指数・着度数P・PCI等）は  
> 一般公開データに含まれないため、実際の重要度は検証してみるまで分かりません。  
> **これが本AIの差別化ポイントです。**

## 10. 特徴量マスタ一覧（説明・重要度・必要理由）

Phase 1（MVP）で使用する全特徴量を横断的に整理した一覧。  
**重要度** は競馬AI先行研究・CrossFactor固有データの希少性・実装コストを加味した推定値。実際の重要度はSHAPで学習後に検証する。

> **重要度凡例:**  
> 🔴 高 = 予測精度へのインパクトが大きい（上位特徴量になりやすい）  
> 🟡 中 = 一定の寄与が見込まれる  
> 🟢 低 = 補助的。単独では弱いが組み合わせで効く

---

### A. レース条件系

| 特徴量名 | 説明 | 重要度 | なぜ必要か |
|---------|------|:------:|-----------|
| `race_distance` | 距離（m） | 🔴 高 | 馬の距離適性は成績に直結する。同じ馬でも1200mと3200mでは全く異なる能力が求められる |
| `track_type` | 芝 / ダート | 🔴 高 | 芝とダートは別競技に近い。芝専門馬・ダート専門馬が存在し、適性が成績を大きく左右する |
| `track_condition` | 馬場状態（良/稍重/重/不良） | 🟡 中 | 重馬場が得意な馬・苦手な馬がいる。特に不良馬場は予測精度の低下原因になりやすいため補正が必要 |
| `weather` | 天候 | 🟢 低 | 馬場状態と相関が高く単独では弱いが、馬場状態との交互作用で補完情報になる |
| `racecourse_id` | 競馬場コード（10場） | 🟡 中 | コース形状・直線距離・坂の有無が競馬場ごとに異なる。得意コース・苦手コースが成績に表れる |
| `race_class` | 競走クラス（G1〜条件戦） | 🔴 高 | クラス間の能力差は大きい。前走クラスと今走クラスの差（昇級・降級）は特に重要な文脈情報 |
| `num_horses` | 出走頭数 | 🟡 中 | 頭数が多いほど展開の不確実性が高まり、穴馬が台頭しやすい。人気馬の連対率に影響する |
| `prize_money_1st` | 1着本賞金（万円） | 🟢 低 | クラス強度を連続値で表せる補助変数。`race_class` のエンコード精度を補完する |
| `straight_distance` | ゴール前直線距離（m） | 🟡 中 | 直線が長いと差し・追込が有利。脚質（`predicted_running_style`）との組み合わせで展開適性を精緻化 |
| `month` | 開催月 | 🟢 低 | 季節による馬場の質・気温・疲労蓄積の違いを捉える。特定の月に強い馬・弱い馬のパターンが存在する |
| `day_of_week` | 曜日（土/日） | 🟢 低 | 日曜の重賞は格が高く出走馬の質が異なる。土曜戦は軽ハンデ・格下レースが多い傾向 |

---

### B. 馬の能力系

| 特徴量名 | 説明 | 重要度 | なぜ必要か |
|---------|------|:------:|-----------|
| `avg_finish_3` | 直近3走の平均着順 | 🔴 高 | 競走馬の直近の状態・能力を最も端的に表す。現在の調子を短期的に捉える主要指標 |
| `avg_finish_5` | 直近5走の平均着順 | 🔴 高 | `avg_finish_3` との差分で「上昇中 / 下降中」のトレンドを把握できる |
| `avg_popularity_3` | 直近3走の平均人気 | 🟡 中 | 実際の着順と人気の乖離（＝市場が過大・過小評価している馬）を発見するために必要 |
| `place_rate_track` | 馬場種別適性スコア | 🔴 高 | 芝 / ダートのどちらに適性があるかを累積実績から定量化。切り替えがある馬の評価に不可欠 |
| `place_rate_course` | 競馬場別適性スコア | 🟡 中 | 特定競馬場への強い得意・不得意は統計的に実在する。コース替わりの影響を事前評価できる |
| `place_rate_distance` | 同距離適性スコア | 🟡 中 | 距離の増減（`distance_change`）とセットで「この距離でどれだけ走れるか」を評価 |
| `win_recovery_rate` | 単勝回収率（過去累積） | 🔴 高 | 人気より着順が安定して良い馬＝期待値が高い馬を識別する。回収率予測と直接連動する希少な特徴量 |
| `place_recovery_rate` | 複勝回収率（過去累積） | 🔴 高 | 複勝の期待値予測に直結。`win_recovery_rate` と合わせて「安定して稼げる馬」の識別に使う |
| `frame_number` | 枠番 | 🟡 中 | 競馬場・距離・馬場状態によって有利な枠・不利な枠が統計的に存在する（内枠有利・外枠不利など） |
| `horse_number` | 馬番 | 🟢 低 | 枠番と相関するが、直線の有利不利に関わる場合もある。補助変数として保持 |
| `horse_weight` | 馬体重（kg） | 🟡 中 | 体重は馬のコンディションバロメーター。体重が軽すぎる / 重すぎる馬は本来の力を出しにくい |
| `horse_weight_diff` | 馬体重増減（kg） | 🔴 高 | 前走比での増減が仕上がり状態を示す。大幅増（+12kg以上）は疲れや太め、大幅減は細め傾向 |
| `carrying_weight` | 斤量（kg） | 🟡 中 | 斤量は直接的に速度に影響する。ハンデ戦では斤量差が最大5〜6kg生じ、成績に大きく影響する |
| `rest_weeks` | 前走からの休養週数 | 🟡 中 | 休養明けは体調・仕上がりが不透明。「0週（ぶっつけ）」「8週以上の長期休養」で成績傾向が変わる |
| `runs_since_return` | 休養後何走目 | 🟡 中 | 長期休養後は2〜3戦かけて本来の力を取り戻す傾向がある。「叩き2走目」等のパターンを学習させる |
| `distance_change` | 前走からの距離増減（m） | 🟡 中 | 急激な距離変更（＋400m以上）は適性未知のリスク。`place_rate_distance` で補完する |
| `horse_age` | 馬齢 | 🟡 中 | 馬の成長・衰退曲線は年齢で表れる。3〜5歳でピーク、7歳以降は一般的に成績が下降する傾向 |
| `horse_sex` | 性別（牡/牝/騸） | 🟡 中 | 牝馬は斤量が2kg軽く有利。ただし牡馬混合戦では牝馬の成績が下がる傾向があるなど補正が必要 |

### C. CrossFactor独自系（差別化の核心）

| 特徴量名 | 説明 | 重要度 | なぜ必要か |
|---------|------|:------:|-----------|
| `position_index` | 位置取り指数（コーナー通過順位の加重平均） | 🔴 高 | 脚質を単純な「逃/先/差/追」の4分類ではなく連続値で表せる。CrossFactor独自。一般データでは再現不可 |
| `position_index_rank` | 位置取り指数の同レース内順位 | 🔴 高 | 絶対値（`position_index`）の同レース内での相対化。「このレースで一番前にいた馬」を特定できる |
| `predicted_running_style` | 予想脚質（逃/先/差/追） | 🔴 高 | 展開が結果を左右する競馬において、各馬の戦い方の予測は不可欠。`straight_distance` との組み合わせで展開有利不利を算出 |
| `top5_4c_rate` | 4角5番手内率（先行安定度） | 🟡 中 | 「いつも前にいられるか」の安定性。逃げ・先行馬の中でも安定して前に行ける馬を識別する |
| `top3_finish_rate` | 上がり3位内率（末脚安定度） | 🟡 中 | 「いつも末脚が切れるか」の安定性。差し・追込馬でも末脚が不安定な馬と安定した馬を区別できる |
| `pci` | ペースチェンジ指数（展開の緩急評価） | 🟡 中 | ハイペース・スローペースを数値化。ペースによって有利不利が変わる馬を識別する重要指標 |
| `turn_aptitude` | 回り適性スコア（左回り/右回り） | 🟢 低 | 右回り得意・左回り苦手な馬は存在する。コース変更時のリスク評価に使用 |
| `weight_correction` | 斤量補正値 | 🟡 中 | 斤量の影響を除いた能力の純粋な比較を可能にする。ハンデ戦での評価精度向上に必要 |
| `weight_correction_finish` | 斤量補正後の上がりタイム補正値 | 🟡 中 | 末脚を斤量の影響を除いて比較。「斤量が重くてもキレる馬」を公平に評価できる |
| `user_index_1` | CrossFactor独自指数1（内容確認中） | 🔴 高（仮） | Shibata様が実運用で参照している指数の可能性が高い。内容確認後に評価を確定する |
