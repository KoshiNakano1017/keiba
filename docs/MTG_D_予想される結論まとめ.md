# MTG 予想される結論まとめ
**作成日:** 2026-03-27  
**目的:** 打ち合わせ前に「こうなるはず」と仮説を持ち、会議をリードするための事前シミュレーション

---

## アジェンダ2｜システム構成・Web・決済（15分）

### 予想される結論

| 論点 | 予想結論 | 根拠・リスク |
|------|---------|------------|
| 技術スタックの承認 | **そのまま承認**（Next.js + Supabase + Stripe） | 既にご提案済みで反論なし |
| 未登録で勝率を見せるか | **一部のみ表示**（上位3頭の勝率だけ見せる等） | 無料ユーザーへの集客と有料転換を両立させたいはず |
| 無料 vs プロの差別化 | **期待値はプロ限定**で合意 | 「最も価値の高い情報」を有料プランに集約するのが収益最大化 |
| プラン変更の即時反映 | **即時反映**を希望 | 解約後も見られると困る、という要望が出るはず |
| Figmaデザインの共有 | **打ち合わせ後に共有**（この場では確認のみ） | 事前準備の余裕がない可能性大 |
| 料金設定 | **未定のまま持ち越し**の可能性高 | ビジネス判断のため Shibata様が検討時間を要する |

### 注意すべきリスク

- 「未登録で全部見えてもいい」と言われた場合 → 収益モデルが崩壊するため、有料転換率の観点から要説明
- 「デザインを完璧に再現してほしい」と言われた場合 → MVPスコープでは対応不可のため、第2フェーズへ先送りを明示

## アジェンダ3｜AIモデル・データ連携（20分）

### 予想される結論

| 論点 | 予想結論 | 根拠・リスク |
|------|---------|------------|
| 最初のモデル | **LightGBMで合意**（スピード優先） | 「3週間で動くもの」の優先度が高いため |
| スキーマ共有 | **打ち合わせ後に共有**（ファイルを送ってもらう） | その場で見せる準備がない可能性 |
| 目標回収率 | **「110%以上を目指したい」レベルで合意** | 具体的な数値は出てきにくいが、方向性は合意できる |
| 賭け種別 | **単勝メインで開始**、その後複数券種に拡張 | まず単純なモデルで検証するのが王道 |
| 再学習頻度 | **月次 or 週次**で合意 | レース頻度（週末中心）に合わせた更新サイクル |
| バックテスト期間 | **3〜5年分**のデータで検証 | CrossFactorの保有データ量次第 |
| 独自データ（映像解析）の扱い | **第2フェーズ以降**で特徴量に追加 | 映像解析データのフォーマット確認が先 |

### 注意すべきリスク

- 「すぐに最高精度のAIを」という期待値ズレ → 「3週間はパイプライン＋ベースライン。精度は継続改善」と最初に宣言
- スキーマが複雑で解読に時間がかかる → 第1週の半分がスキーマ理解に消える可能性を折り込む

## アジェンダ4｜スケジュール・進め方（10分）

### 予想される結論

| 論点 | 予想結論 | 根拠・リスク |
|------|---------|------------|
| 3週間スケジュール | **おおむね合意** | すでに提案済みのスケジュール通り |
| 連絡手段 | **Slack or チャットワーク**で合意 | フリーランス案件の標準ツール |
| 報告頻度 | **週2回**（月・木の進捗共有） | 週1では遅すぎ、毎日では負荷が高い |
| Shibata様側の宿題 | **SQLiteサンプル・Figma・Stripe準備**をお願い | ここが遅れると第1週が止まるため最優先 |
| 支払い条件 | **完成後一括 or 中間50%＋完成後50%** | 金額220,000円のため2分割を好まれる可能性あり |

### 注意すべきリスク

- SQLiteサンプルの共有が遅れた場合 → 第1週の着手が遅れ、3週間完遂が困難になるため期限（4/1）を明示
- Stripeアカウント未作成の場合 → 第2週に影響。早めに準備依頼する
