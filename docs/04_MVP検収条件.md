## 1. MVPの定義（スコープ）

### MVP範囲内（確定）
- CrossFactor（SQLite）→ Supabase へのデータ自動同期
- AIモデルによる予想勝率の算出
- Next.js製Webサイトでの予測結果表示
- Stripe決済によるサブスクリプション管理
- プラン別のデータアクセス制御（RLS）

### MVP範囲外（後回し・確定）
- ユーザー向けの詳細UI（Figma完全再現は第2フェーズ）
- 管理画面（Shibata様用のデータ操作UI）
- モバイルアプリ
- AI精度の最終チューニング（初期はベースラインモデルで可）

### MVP範囲（未定）
- 地方競馬データの対応 | 未定（Shibata様確認後）
- 無料プランの表示範囲の詳細 | 未定（権限マトリクス確定後）

## 2. 週次マイルストーンと検収基準

### 第1週末（3週間後 -14日）: データ基盤の完成
**確認事項（検収条件）**

| # | 確認項目 | 合否基準 |
|---|---------|---------|
| 1 | CrossFactor SQLiteからデータ抽出できる | Python スクリプトが正常終了し、CSVまたはSupabaseに出力される |
| 2 | Supabaseへの自動アップロードが動作する | タスクスケジューラ実行で Supabase テーブルにデータが入る |
| 3 | AIモデルのベースラインが動作する | 指定データで予想勝率が出力されるノートブックが存在する |

### 第2週末（3週間後 -7日）: 決済・認可の完成
**確認事項（検収条件）**

| # | 確認項目 | 合否基準 |
|---|---------|---------|
| 1 | Stripeでサブスクリプション登録できる | テストカードで決済が完了し、プロプランに切り替わる |
| 2 | プラン変更が即時反映される | Webhookイベント受信後、Supabase上のプランカラムが更新される |
| 3 | RLSによる権限制御が動作する | 未登録ユーザーが制限データを取得できないことをAPIレベルで確認 |

### 第3週末（最終週）: 結合テスト・本番デプロイ
**確認事項（最終検収条件）**

| # | 確認項目 | 合否基準 | ステータス |
|---|---------|---------|----------|
| 1 | Windows → クラウド → Web の一気通貫動作 | データ更新がWebに反映されることをE2Eで確認 | **確定（方針）** |
| 2 | 予測結果がWebに表示される | 予想勝率がプロプランで表示される | **確定（方針）** |
| 3 | Vercelへの本番デプロイ完了 | https://（本番URL）でアクセス可能 | **確定（方針）** |
| 4 | 操作手順書（引き継ぎ資料）の納品 | Python スクリプトの実行手順、Supabase操作、Stripe管理手順を記載 | **確定（方針）** |
| 5 | 回収率バックテスト結果の提示 | 未定（目標ラインは打ち合わせで合意） | 未定 |

## 4. 「支払い対象」の定義

> 以下を「支払い対象（検収完了）」とする

1. 上記「第3週末の最終検収条件」の必須項目がすべて合格であること
2. 実際の サーバーがわ の Windows 環境で一気通貫動作が確認できること
3. バックテスト結果が「未定の目標ライン」に準ずる合理的な説明ができること

> **注意:** AI精度の「絶対値」を検収条件にすると、初回は達成困難なため、  
> 「動くシステム＋説明可能なモデル＋バックテスト結果の提示」を検収条件とすることを推奨する。
