## 1. CrossFactor → AI → Web のデータフロー

```
CrossFactor（SQLite）
  │
  ├─ テーブル群（レース結果 / 馬情報 / 調教 / 映像解析データ）
  │
  └─ [Python: 抽出スクリプト]
        │
        ├─ 特徴量生成（前処理・エンジニアリング）
        │
        ├─ AIモデル推論（学習済みモデルをロード）
  │     └─ 出力: 予想勝率・期待値など
        │
        └─ Supabase API へアップロード
              └─ predictions テーブルに保存
                    └─ Next.js が読み取り → Web表示
```

| 独自データ | 意味 | 活用方針 |

## 3. AIモデル設計

### ①フェーズ別モデル戦略

| フェーズ | モデル | 目的 | 期間 |
|---------|-------|------|------|
| **Phase 1（MVP）** | LightGBM | ベースライン構築・動作確認 | 第1週 |
| **Phase 2** | CatBoost / XGBoost | アンサンブル・精度向上 | 第2週以降 |
| **Phase 3** | Neural Network（TabNet等） | 非線形パターン捕捉 | 改善フェーズ |

> **なぜLightGBMから始めるか**  
> ・学習速度が速く、3日以内にベースライン構築可能  
> ・特徴量重要度が可視化でき、Shibata様との議論がしやすい  
> ・競馬AIの先行研究でも高性能が確認されている

### ②入力データ（特徴量）の候補

**馬の能力系**
- 近5走の着順・タイム・上がりタイム
- 能力指数（CrossFactor独自）
- 距離適性・コース適性
- 前走からの距離変化

**レース条件系**
- 距離・芝/ダート・馬場状態
- クラス・賞金
- 枠番

**騎手・調教師**
- 騎手の近走勝率・コース別成績
- 調教師の近走勝率

**調教データ**
- 調教タイム・本数・強さ

**CrossFactor独自（差別化）**
- 前走の距離ロス補正後タイム
- 前走の不利有無・度合い
- トラックバイアス補正値
- 走行ポジション（先行/差し/追込）

**オッズ系**
- 単勝オッズ（締め切り直前）
- 人気順

### ③出力定義（確定）

| 出力 | 説明 | 優先度 |
|------|------|-------|
| 予想勝率（%） | 1着になる確率 | **必須** |
| 予想連対率（%） | 2着以内に入る確率 | 可能であれば |
| 予想複勝率（%） | 3着以内に入る確率 | 可能であれば |
| 期待値 | 予想勝率 × 単勝オッズ | 可能であれば |

## 4. バックテスト設計（評価方法）

### データ分割方針

```
時系列で厳密に分割（リーク防止）

例:
  学習データ  : 〜2023年12月
  バリデーション: 2024年1月〜6月
  テスト（評価）: 2024年7月〜（未来データ）

※ランダム分割は禁止（時系列の場合、過去データで未来を予測することになる）
```

### 回収率計算ルール

| ルール | 内容 | ステータス |
|-------|------|----------|
| オッズ取得時点 | 締め切り直前のオッズ（事後オッズ禁止） | **確定** |
| JRA控除率 | 約25%を反映 | **確定** |
| 賭け種別 | 未定 → **打ち合わせで確定** | 未定 |
| 賭け金配分 | 均等 / ケリー基準 / その他 → **打ち合わせで確定** | 未定 |
| 目標回収率 | 未定 → **打ち合わせで確定** | 未定 |
